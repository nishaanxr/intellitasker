\begin{abstract}
The rapid growth of digital tools has transformed the way software development squads collaborate and coordinate tasks. However, developers still rely on fragmented tools, including standard task lists, spreadsheet sheets, and separate chat applications, for project-related coordination. This creates several challenges, such as notification fatigue, high cognitive friction when updating task states, a lack of centralized task context, and a general avoidance of project tracking systems. The IntelliTasker is proposed as a centralized, web-based task management platform that brings together task visualization, dynamic subtask decomposition, secure user authentication, and real-time collaboration in a unified ecosystem. The platform enables users to manage tasks through custom board and list layouts, generate dynamic action plans using generative AI models, protect metadata through clock-drift robust two-factor authentication (2FA), and maintain focus using a dedicated calm workspace with integrated focus audio. The platform is developed using React, Tailwind CSS, Express, Node.js, and MongoDB Atlas. It follows a modular and scalable architecture that ensures better performance, ease of maintenance, and future extensibility. The proposed solution addresses the limitations of traditional task managers by combining visual coordination, AI chatbot assistance, developer well-being features, and robust security into a single application.
\end{abstract}

\section{Introduction}
Project and task management have become critical components of modern software engineering workflows. Software development teams rely heavily on digital systems to coordinate tasks, assign responsibilities, track project timelines, and document development history. However, despite the widespread availability of software coordination tools, teams still face significant difficulties in managing their workloads efficiently.

Currently, developers and teams depend on multiple fragmented platforms to organize their activities. Teams use social messaging applications for general discussion, spreadsheet software to log progress updates, external media applications for workspace music, and complex enterprise task boards to log formal subtasks. This fragmented approach leads to poor communication, a lack of centralized contextual information, high administrative overhead, and decreased user engagement with project tracking systems.

Existing project tracking tools such as Trello, Jira, and Notion provide partial support for task management, scheduling, and documentation. However, these tools are limited because they do not offer all the necessary features in a single, unified environment. For example, Jira supports team workflows but requires high administrative effort and manual forms, Notion is highly customizable but lacks native context-aware AI agents, and Trello provides simple visual boards but lacks built-in well-being features or focus environments.

The IntelliTasker platform is proposed to overcome these limitations by creating a centralized and integrated platform for collaborative task management. The platform is designed to connect team members and project administrators through task-specific visual boards, interactive conversation modules, secure session management, and calm work mode components.

The platform provides users with the ability to create structured tasks with priorities, assignees, deadlines, and descriptions. Standard members can view and manage their assigned tasks, while administrators retain full project visibility and can filter tasks by individual team members to review workloads. A custom AI chatbot allows users to query their active tasks, discuss progress, and automatically break down complex deliverables into actionable subtasks. Additionally, a dedicated Calm Mode minimizes distractions and integrates an ambient lo-fi music player to foster a productive workspace.

The proposed system is developed using modern web technologies, including React, Tailwind CSS, Node.js, Express, and MongoDB. These technologies provide scalability, security, performance, and flexibility for future improvements. The modular design of the application makes it easier to maintain and extend with advanced features in the future, such as analytical dashboard screens, multi-project boards, and automated report generators.

Overall, the IntelliTasker platform aims to build a complete digital ecosystem for task management that reduces tool fragmentation and administrative friction while improving the overall developer experience.

\section{Literature Review}
Digital project management systems have changed the way development squads organize workloads and coordinate milestones. Software professionals, system administrators, and project managers rely heavily on software interfaces for task scheduling, roster coordination, and team updates. However, most of the currently available tools are designed for single, isolated purposes, such as static database tracking, external messaging, or simple documentation. This creates a gap in collaborative team environments because users must context-switch between several disconnected applications to complete simple task updates.

Collaborative visual task managers focus primarily on task cards, progress columns, and state transitions. These systems simplify progress tracking and help team members visualize work statuses. Although they are useful for general status overviews, they require considerable manual administration, including form filling, text descriptions, and drop-down selections. This manual effort can discourage users from keeping task boards updated, leading to outdated project tracking records.

Conversational interfaces and chatbot assistants have emerged as effective tools for reducing software interaction friction. By utilizing natural language processing, chatbots allow users to perform database operations and query records through simple text prompts. However, standard chatbot integrations typically act as external panels without access to the actual workspace database context. This means the chatbot cannot answer specific project-related queries unless the user manually copy-paste relevant task descriptions.

Developer fatigue and cognitive overload have also become key areas of interest in modern software research. Studies have shown that constant notifications, alert bells, and visual clutter from project dashboards contribute to notification fatigue and disrupt focus. Calm technology principles suggest that software should fit into the periphery of user attention rather than constantly demanding focus. Research also shows that ambient auditory stimuli, such as lo-fi music, can help developers maintain their attention span and reduce workspace anxiety. Despite these benefits, standard task boards do not include focus modes or built-in media controls.

Security also remains a critical requirement for project management databases, as they house sensitive system details, code milestones, and credentials. While multi-factor authentication (MFA) is commonly recommended, time-based one-time password (TOTP) algorithms often fail due to clock drift between client mobile devices and hosting servers. This drift can lock users out of their accounts, highlighting the need for resilient verification windows.

Several researchers have analyzed the role of digital communication tools in modern engineering squads:
\begin{itemize}
  \item Studies show that integrating central messaging setups with task managers improves project transparency and team trust \cite{ref1}.
  \item Other researchers highlight that automated visual dashboards help managers identify workflow blockages and allocate resources effectively \cite{ref2}.
  \item Further research demonstrates that conversational user interfaces reduce the time required to log task progress, improving data accuracy on project boards \cite{ref3}.
  \item Analysis of developer work habits suggests that minimizing UI clutter and notification noise significantly extends sustained focus periods \cite{ref4}.
  \item Studies in software well-being confirm that embedding ambient focus music inside workspace applications helps developers enter a state of deep focus more easily \cite{ref5}.
  \item Research on time-based authorization shows that adjusting validation windows in authentication servers reduces user lockouts caused by clock drift \cite{ref6}.
  \item Security analyses of development platforms emphasize that role-based access control prevents unauthorized members from accessing sensitive database records \cite{ref7}.
  \item Studies in agile task tracking conclude that dynamic, real-time board updates are essential for maintaining accurate coordination in fast-paced software projects \cite{ref8}.
\end{itemize}

The literature reviewed above confirms that while individual tools address specific issues like task tracking, authentication, or workspace focus, no single system combines a collaborative task board, an AI-powered conversational chatbot, calm focus environments, and clock-drift resilient security in one integrated MERN application. Therefore, the proposed IntelliTasker aims to address these limitations by providing a unified, low-friction platform for modern development squads.

\section{Methodology}
The proposed IntelliTasker platform follows a modular and user-centric approach to provide a centralized task management environment. The system is designed to allow users to register securely, complete 2FA setup, manage task boards, decompose complex milestones with AI assistance, interact through a conversational chatbot, and maintain focus in a calm mode workspace. The development methodology is structured into four distinct phases, as illustrated in the methodology flow represented in Figure \ref{fig:methodology}.

\subsection{Phase 1: Requirement Analysis}
The initial phase focuses on defining the system parameters and understanding the practical challenges faced by developers using traditional task managers. The primary stakeholders of the platform include project administrators, software developers, and system coordinators. Their requirements are gathered by reviewing the bottlenecks in existing task coordination workflows.

Functional requirements such as secure login, 2FA setup, interactive Kanban layouts, conversational chatbot querying, automated subtask generation, user profile views, and task comments are defined. Non-functional requirements, including page loading speeds, password hashing strength, system reliability, visual consistency, and screen responsiveness, are also identified.

\subsection{Phase 2: System Design and Planning}
In the second phase, the application's architecture is planned. The system is split into three decoupled tiers: the presentation frontend layer, the application logic backend layer, and the database storage layer. The frontend is designed as a single-page application using React, TypeScript, and Tailwind CSS. The backend is designed as an Express server running on Node.js, and the database utilizes MongoDB with Mongoose schemas.

Database structures are planned to store users, profiles, tasks, subtask lists, comment cards, and historical trails. Visual wireframes are created to design a dark dashboard interface. Security planning is also completed, focusing on JSON Web Token (JWT) handling, password hashing, and Speakeasy 2FA settings to prevent unauthorized database modifications.

\subsection{Phase 3: Development and Integration}
The development phase involves coding the independent modules of the application and subsequently integrating them to ensure smooth communication. The interaction between users and these major platform modules is illustrated in Figure \ref{fig:interaction}.

\begin{figure}[!ht]
\centering
\includegraphics[width=0.48\textwidth]{interaction}
\caption{Interaction Diagram representing user authorization traversal, workspace navigation pathways, Calm Mode activation, and background database synchronization loops.}
\label{fig:interaction}
\end{figure}

The platform comprises several core modules:
\begin{enumerate}
    \item \textit{User Authentication and Speakeasy 2FA Module:} Manages registration and secure login. It integrates the Speakeasy library to generate shared secrets and verify TOTP tokens, using an expanded validation window to remain resilient to clock drift.
    \item \textit{Task Board and Kanban Module:} Provides an interactive, clean interface to display tasks under "To Do" (pending), "In Progress" (in-progress), and "Done" (completed) states.
    \item \textit{AI Conversational Chatbot Module:} Embeds a chat panel directly in the workspace. It dynamically passes the active user's pending tasks to the Google Gemini model as system context, allowing the chatbot to answer specific project-related questions.
    \item \textit{AI Action Plan Module:} Decomposes complex tasks into actionable subtasks with a single click, automatically parsing the AI output and saving it to the task schema.
    \item \textit{Calm Mode Subsystem:} Hides distracting visual panels and integrates a lo-fi ambient audio player to help developers maintain focus.
    \item \textit{Role-Based Task visibility Module:} Automatically restricts standard members to seeing only their own assigned tasks, while providing admins with a member filter dropdown to view individual workloads.
    \item \textit{Collaboration and Comments Module:} Enables team members to post comments and coordinate updates on individual tasks.
    \item \textit{Task History Trail Module:} Automatically logs state changes, updates, and milestones on each task to maintain a clear audit trail.
\end{enumerate}

\subsection{Phase 4: Testing and Validation}
Following development, the system undergoes systematic testing to confirm its functionality and local performance. Unit tests are run on key components, including the 2FA verify route, Mongoose schema validation, and context extraction functions. Integration testing verifies that updates made via the AI chatbot or task board are saved correctly in the database.

System testing verifies that the dashboard updates dynamically under various browser configurations. Finally, user acceptance tests are conducted with a small group of developers to collect feedback on ease of use, task card readability, and the effectiveness of the Calm Mode environment.

\section{Requirement Specification}
This section defines the functional and non-functional requirements of the proposed IntelliTasker platform.

\subsection{Functional Requirements}
Functional requirements define the core operations the application must perform:
\begin{enumerate}
    \item \textit{User Access Requirements:}
    \begin{itemize}
        \item Users must be able to register and log in securely.
        \item The system must support two-factor authentication (2FA) setup via QR codes and Speakeasy verification.
        \item The authentication route must accept valid TOTP tokens within an expanded time window to prevent clock-drift lockouts.
    \end{itemize}
    \item \textit{Task Board and Layout Requirements:}
    \begin{itemize}
        \item The dashboard must render tasks in dynamic Kanban board and tabular list layouts.
        \item Standard members must only see tasks assigned to them on their boards and in their metrics.
        \item Administrators must see all tasks by default and have access to a member filter dropdown to view individual workloads.
        \item Completed tasks must not render warning borders or alert styles, even if marked as high priority.
    \end{itemize}
    \textit{AI Chatbot and Automation Requirements:}
    \begin{itemize}
        \item The chatbot must accept user prompts and respond using context-injected task lists.
        \item The AI Action Plan feature must break down tasks into subtasks and save them directly to the database.
        \item The system must clean markdown code formatting from the AI response before saving the list.
    \end{itemize}
    \textit{Calm Mode and Sensory Isolation Requirements:}
    \begin{itemize}
        \item The UI must allow users to enter a distraction-free mode.
        \item Entering Calm Mode must hide sidebars, filter rows, and distracting elements, showing only the active task.
        \item Calm Mode must integrate an ambient lo-fi music player with play, pause, and volume controls.
    \end{itemize}
\end{enumerate}

\subsection{Non-Functional Requirements}
Non-functional requirements define the quality and usability standards of the system:
\begin{enumerate}
    \item \textit{Performance Requirements:}
    \begin{itemize}
        \item The web dashboard should load quickly and update task cards with minimal delay.
        \item Database reads and writes must execute efficiently using index optimizations.
        \item Conversational AI responses should compile and render within a reasonable timeframe.
    \end{itemize}
    \item \textit{Security Requirements:}
    \begin{itemize}
        \item Password storage must use strong BCrypt hashing.
        \item API endpoints must be protected using JWT verification.
        \item Client-server communication must occur over secure channels.
    \end{itemize}
    \item \textit{Scalability and Maintainability Requirements:}
    \begin{itemize}
        \item The system architecture must be modular to support new modules or features.
        \item The MongoDB database schema must accommodate future collections and fields without requiring major rewrites.
    \end{itemize}
    \item \textit{Usability Requirements:}
    \begin{itemize}
        \item The interface must offer a clean, premium dark theme.
        \item Page layouts must be responsive across desktops, tablets, and mobile devices.
        \item Interactive elements must be easy to locate and use.
    \end{itemize}
\end{enumerate}

\section{Design Approach}
The proposed IntelliTasker platform utilizes a layered, modular design approach to ensure scalability and ease of maintenance, separating frontend interfaces, business logic, and databases.

\begin{figure}[!ht]
\centering
\includegraphics[width=0.48\textwidth]{architecture}
\caption{System Architecture of the IntelliTasker Platform illustrating the decoupling between the React.js client interface, Node.js/Express.js REST controllers, persistent MongoDB schemas, and Google Gemini AI orchestration layer.}
\label{fig:architecture}
\end{figure}

\subsection{Frontend Layer}
The frontend layer is responsible for the user interface and interactions. It is built as a single-page application using React, TypeScript, and Tailwind CSS. The frontend manages the application state, routes views, and coordinates real-time user updates. Key interfaces include the central Kanban board, the list layout, the secure login screen with 2FA inputs, the sliding conversational chatbot panel, and the minimalist Calm Mode focus container.

\subsection{Backend Layer}
The backend layer processes the business logic, API requests, and communication between the client and database. Developed as a Node.js and Express server, the backend handles JWT verification, manages user registration, and houses task update routes. It also handles external API calls, retrieving active task lists and passing them as context to the Google Gemini model.

\subsection{Database Layer}
The database layer manages data storage and retrieval. PostgreSQL is used in the reference system, but IntelliTasker uses MongoDB Atlas, a fully managed cloud database service, with the Mongoose ODM for persistent storage. This document-based cloud database structure is well-suited for collaborative task managers, as it allows subtasks, task histories, and comments to be stored as nested document arrays within the primary task collections. This approach simplifies the database schema and reduces read latencies by avoiding complex relational joins.

\subsection{Modular Design}
The system is divided into independent modules, allowing features to be updated without affecting other parts of the application:
\begin{itemize}
    \item \textit{User Access and Security Module}
    \item \textit{Task board Dashboard Module}
    \item \textit{Conversational AI Chatbot Module}
    \item \textit{AI Subtask Decomposition Module}
    \item \textit{Calm Mode Focus Module}
    \item \textit{Role-based Task Visibility Module}
    \item \textit{Task Comments and History Module}
\end{itemize}

\subsection{Security Design}
Security is integrated throughout the application's design. Sensitive routes are protected by verification tokens, and user credentials are secured with BCrypt hashing. Setup and validation of 2FA secrets are managed through Speakeasy, using adjusted verification windows to ensure that minor time discrepancies on user devices do not prevent successful authentication.

\subsection{Scalability and Maintainability}
By utilizing a modular MERN stack architecture, the system is designed to support future expansions. Developers can integrate new modules, such as administrative report generators, multi-project workflows, or external repository integration, with minimal changes to the core system.

\section{Results}
The developed IntelliTasker platform was successfully implemented, with all core modules functioning as planned. The application provides an integrated, low-friction environment for managing collaborative task boards, interacting with AI models, and maintaining developer focus.

The secure access module allows users to register, log in, and complete 2FA setup. The Speakeasy 2FA verification system successfully validates authentication tokens. Even when user devices have minor clock differences (up to 4 minutes), the system accepts the input, preventing user lockouts.

The task board successfully displays active tasks in both Kanban board and tabular list layouts. Standard team members see only their assigned tasks, while project administrators have full visibility over all active tasks. The admin member filter dropdown allows administrators to select any member and view their active tasks, reducing task tracking complexity. Completed tasks do not render warning borders or warning alerts, keeping the interface clean. The pending task metric correctly counts tasks in both "To Do" and "In Progress" columns.

The conversational AI chatbot compiles responses using context-injected task lists, successfully answering questions about active tasks, priorities, and deadlines. The AI Action Plan feature successfully decomposes complex tasks into actionable subtasks with a single click, cleaning markdown formatting and saving the subtasks directly to the MongoDB database.

The Calm Mode workspace successfully isolates the user, hiding sidebars and other distracting visual panels. The integrated lo-fi music player functions as intended, allowing users to play focus music directly within the application.

Testing confirmed that the platform performs efficiently under normal conditions. Page loading times are minimal, task state transitions update smoothly, and database operations execute without significant delay. The MERN architecture successfully integrates the visual task board, AI chatbot assistance, calm mode, and secure 2FA verification into a single, cohesive web application.

\section{Conclusion}
The IntelliTasker platform was developed to address tool fragmentation and administrative friction in software development squads. Traditional task managers often require significant manual effort and lack focus-assisting features or secure, integrated conversation tools.

The proposed system successfully integrates a collaborative task board, a context-aware AI chatbot, dynamic subtask decomposition, a distraction-free Calm Mode workspace, and clock-drift resilient 2FA security into a single application. Developed using React, Tailwind CSS, Node.js, Express, and MongoDB, the platform offers good scalability, performance, and security.

The results demonstrate that the platform provides a unified task management environment that can reduce administrative friction, enhance developer focus, and improve data transparency. The modular design of the application allows for future enhancements, such as integration with version control systems, multi-project workflows, and performance analytics. In conclusion, the proposed IntelliTasker platform provides a robust and practical solution for modern software development squads.
